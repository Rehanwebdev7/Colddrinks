## 5. Kya kya try kiya — aur kyu fail hua

| Approach | Result | Reason |
|---|---|---|
| Firebase auto-created Service Account | ❌ Failed | Firebase-managed SAs have stricter quotas |
| Spring Boot SA (cross-account) | ⚠️ Apparently worked but unreliable | Different Google account ki SA — risky dependency |
| Standalone GCP project SA (`cold-drinks-drive`) | ❌ Failed | Google's 2024 policy: **NO service account** can upload to personal Drive folders. `storageQuotaExceeded` error |
| Workspace Shared Drive | Skipped | Paid plan chahiye, user ne free chaha |
| **OAuth Refresh Token** | ✅ **Working** | Files owned by human user → 15 GB free quota use hota hai |

**Lesson:** Personal Gmail Drive ke liye SA bilkul kaam nahi karta — sirf OAuth refresh token route hai.

## 6. Token Generate Karne Ka Tareeka (Future me agar dobara karna pade)

### Kab dobara karna padega?

| Situation | Action |
|---|---|
| Backend logs me `invalid_grant` aaye | Token revoke ho gaya — regenerate |
| https://myaccount.google.com/permissions par jaake "Cold Drinks Drive Backend" remove kar diya | Regenerate |
| 6+ mahine app use nahi kiya | Regenerate |
| OAuth Consent Screen delete kar diya | Pure setup dobara (Step 4 onwards) |
| Password change kiya | Regenerate **NAHI** karna — token surive karta hai |

### Regenerate Steps

```bash
cd backend
node scripts/generate-drive-token.js
```

1. Browser khulega
2. `rehanwebdev7@gmail.com` se login (yeh **CRITICAL** hai — folder owner same hona chahiye)
3. "Google hasn't verified this app" → Advanced → Go to Cold Drinks Drive Backend (unsafe) → continue
4. Allow click karo
5. Browser pe green "Drive token saved" dikhega → tab close
6. `backend/drive-credentials.json` overwrite ho gaya
7. **Backend restart kar** — nahi to purana cached token use karega

## 7. Troubleshooting Checklist

### Issue: Image upload pe 500 error

| Check | How |
|---|---|
| Backend running hai? | `netstat -ano \| findstr :8000 \| findstr LISTENING` |
| Latest credentials load hua hai? | Backend restart kar (Node `require()` JSON cache karta hai) |
| Helper directly se kaam kar raha? | `node -e "require('./helpers/drive').health().then(console.log)"` — agar `rehanwebdev7@gmail.com` aaye to OK |
| Frontend admin se logged in hai? | DevTools → Application → Cookies → admin token check |
| Network tab me actual error kya hai? | DevTools → Network → `/api/drive/upload` → Response tab |

### Issue: `storageQuotaExceeded`

| Check |
|---|
| `drive-credentials.json` me `"type": "oauth_user"` hai? Agar `"type": "service_account"` hai to wo galat file hai — token generator chala |
| Sahi account se login kiya tha? `rehanwebdev7@gmail.com` (folder owner). Agar koi aur Gmail use kiya to wo account ka quota use karega — folder access nahi hoga |

### Issue: Image delete kiya but Drive me reh gaya

| Check |
|---|
| Backend latest version chal raha? `extractDriveFileId` aur `deleteDriveFilesFromUrls` functions code me hain? |
| Backend logs me `[drive] cleanup failed` warning aaya? Agar haan to actual error wahin pe hai |
| URL format `https://lh3.googleusercontent.com/d/XXX` hai? Agar koi aur format hai to regex match nahi hoga |

### Issue: `Port 54321 in use` (token generator)

| Action |
|---|
| Pichla generator process abhi bhi chal raha hoga: `netstat -ano \| findstr :54321` → kill kar |
| Ya script me `PORT` 54321 se badal ke 54322 kar de temporarily |

### Issue: `No refresh_token` (token generator)

| Reason | Fix |
|---|---|
| Pehle bhi authorize kiya tha is app ko | https://myaccount.google.com/permissions → "Cold Drinks Drive Backend" remove kar → script dobara chala |

### Issue: Browser me "App not verified" warning

Yeh **NORMAL** hai. App Testing mode me hai isliye warning aati hai. Click sequence:
1. **Advanced** click (chhote text me)
2. **Go to Cold Drinks Drive Backend (unsafe)** click
3. Continue with consent

Yeh safe hai kyunki tu khud ka app hai.

## 9. Critical Reminders

| Reminder | Why |
|---|---|
| **OAuth app NEVER publish karna** | Testing mode me refresh token forever valid hai (drive.file non-sensitive scope) — Publish karne se kuch fayda nahi |
| **Login hamesha `rehanwebdev7@gmail.com` se** | Folder owner same hai, koi aur account use kiya to upload fail hoga |
| **Backend restart after credentials change** | Node `require()` cache karta hai — purana token use karega warna |
| **`oauth-client.json` aur `drive-credentials.json` git me kabhi nahi** | Dono me secrets hain |
| **Frontend me Drive ka koi UI nahi** | Admin/customer ko silent rakha hai — agar future me badge add karna ho to `useDrive` hook ka stub modify kar |
| **Image cleanup automatic hai** | Product/Slider delete ya image replace pe Drive bhi clean ho jata hai (server.js ke `deleteDriveFilesFromUrls` se) |

## 12. Final Architecture Decision Log

| Decision | Reason |
|---|---|
| **OAuth, not Service Account** | Google's 2024 policy blocks SA upload to personal Drive — only OAuth works |
| **`drive.file` scope, not `drive`** | Non-sensitive → no verification, no expiry, least-privilege |
| **Testing mode, not Published** | drive.file me Published se koi fayda nahi, Testing simpler |
| **Desktop app OAuth client, not Web** | Loopback redirect easier, no domain verification |
| **Frontend silent (no Drive UI)** | Admin/customer ko Drive ka pata nahi — backend transparently handle karta hai |
| **Backend cleanup on every DELETE/UPDATE** | Drive me orphans nahi banne chahiye — Firestore source of truth |
| **Public read permission on uploads** | Image `<img src>` me direct render ho — signed URL overhead nahi |
| **base64 over JSON, not multipart** | Existing API patterns ke saath consistent — simpler |
